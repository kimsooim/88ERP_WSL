버전 이름(태그)은 SemVer(Semantic Versioning) 기준으로 해:

  v[Major].[Minor].[Patch] 형식

  Major (첫 번째 숫자)
  - 큰 변화, 호환성 깨지는 변경
  - 예: v1.0.0 → v2.0.0 (전체 구조 변경)

  Minor (두 번째 숫자)
  - 새 기능 추가, 하위 호환성 유지
  - 예: v1.0.0 → v1.1.0 (새 페이지 추가)

  Patch (세 번째 숫자)
  - 버그 수정, 작은 개선
  - 예: v1.1.0 → v1.1.1 (오타 수정)

  우리 프로젝트 기준:

  - v1.0.0: 웹대시보드 기본 구조 완성 (첫 안정 버전)
  - v1.1.0: 전체 48개 페이지 완성 (기능 추가)
  - v1.1.1: 버그 수정이나 작은 개선
  - v1.2.0: 자동화 기능 추가 시
  - v2.0.0: 전체 아키텍처 변경 시

  언제 버전을 올려야 할까:
  - 주요 기능 완성할 때마다 Minor 버전 업
  - 중요한 백업 시점마다 태그 생성
  - 운영 배포 전에는 반드시 태그 생성
