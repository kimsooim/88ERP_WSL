  🚀 88ERP 자동 배포 시스템

  언제 실행되나?

  - main 브랜치에 푸시할 때마다 자동 실행
  - 수동으로도 실행 가능 (GitHub Actions에서)

  무엇을 하나?

  1. 개발 → 운영 코드 동기화
    - 개발 저장소: 88ERP_WSL (지금 작업하는 곳)
    - 운영 저장소: 88ERP_server (서버용 깨끗한 버전)
    - 웹대시보드 코드만 복사 (필요한 파일만)
  2. 운영 서버에 배포
    - 서버 위치: server 컴퓨터 (회사 노트북)
    - SSH로 서버 접속
    - Docker 컨테이너 재시작
    - 웹사이트 업데이트 완료!

  왜 이렇게 하나?

  - 자동화: 코드 수정하고 푸시만 하면 자동으로 서버에 반영
  - 안전성: 개발 코드와 운영 코드 분리
  - 편의성: 복잡한 배포 과정을 자동으로 처리

  현재 문제

  - PROD_REPO_TOKEN이 없어서 88ERP_server 저장소에 접근 못함
  - 그래서 오류 메일이 온 거야
