## 88ERP 오프라인사업 자동화 페이지 개발 완료
날짜: 2025-07-01
작업자: Ann & Claude Code

### 개요
웹대시보드 오프라인사업 > 자동화 페이지 UI/UX 전면 개편 완료
- 거래처별 자동화 워크플로우 시각화
- n8n 워크플로우 연동 준비
- 실시간 진행 상태 표시 기능 구현

### 주요 작업 내용

#### 1. 거래처별 자동화 워크플로우 구현
- 에이랜드 (일일매출/월말매출) 분리
- 교보문고 (일일매출/월말매출) 분리
- 일상의틈, 모닝글로리, 닷밀 등 거래처별 워크플로우 설정
- 각 거래처별 데이터 경로 표시 (예: \\ds920\2bot\2aland\input)

#### 2. UI/UX 개선사항
- 2열 그리드 레이아웃으로 변경
- 버튼 가시성 문제 해결 (시작/중지/새로고침 항상 표시)
- 상태 표시를 버튼 앞쪽으로 이동 (레이아웃 안정화)
- 버튼 색상 통일 (비활성화 시 bg-gray-100)
- 완료 상태 색상 변경 (녹색 → 파란색)

#### 3. 워크플로우 단계별 정보 표시
- 비즈패션 데이터 다운: 다운로드된 행 수 표시 (지점수 × 120행)
- 엑셀 데이터 분석: 생성된 파일명 표시
- 이메일 발송: 수신자 이메일 주소 표시 (ann@88toy.co.kr)
- Supabase 전송: 전송된 데이터 행 수 표시 (지점수 × 45행)
- NAS 데이터 저장: 저장된 데이터 행 수 표시 (지점수 × 80행)

#### 4. 레이아웃 최적화
- 마지막 실행 날짜와 데이터 경로를 같은 줄에 배치
- 데이터 경로 오른쪽 정렬
- 진행률 정보 카드 하단 오른쪽 정렬
- 고정 높이 설정으로 레이아웃 움직임 방지

#### 5. 진행 상태 기능
- 실시간 지점 수집 현황 표시
- 단계별 진행 바 애니메이션
- 완료 시 수집된 파일 경로 표시

### 기술 스택
- Next.js 14.0.4
- React (useState hooks)
- Tailwind CSS
- TypeScript
